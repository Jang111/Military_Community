*데이터 베이스 생성 중 시행착오
1. MYSQL을 사용하기 위해서는 Django 파일중 settings.py의 DATABASES 프롬프트를 수정하여야 했음
2. pyhton manage.py migrate 명령어를 cmd창에 입력하여 테이블을 생성하려 했으나  Can't connect to server on '127.0.0.1'(10061)")에러 발생
-> MYSQL 및 Workbench가 설치되지 않았던 것
3. MYSQL 설치 수행 -> 에러 발생-> Visual c++ 설치가 안되어 있었음 -> visual c++설치
4. 다시 mysql 및 workbench 설치
5. 가상환경에서 pip install mysqlclient 명령어를 통해 mysqlclient 설치
6. django unknown database name에러 발생
-> 이 에러는 알고보니 말그대로 데이터베이스가 아직 생성되지 않았기에 발생하는 오류였음
-> cmd 창에 mysql 실행 후 create database community;(community는 db이름이다.)라고 입력한후
'\q'를 통해 mysql을 종료한 후 python manage.py migrate 명령어를 수행하였더니 원활히 작동함

**
pip install mysqlclient가 먹히지 않을 때는 python 버전을 낮춰주는 것도 방법일 듯하다.
(나같은 경우 3.12를 쓰다가 3.8로 바꿨더니 바로 됨..)
